## Root Cause Analysis

The verification failure `sh: 1: cyrus2/.venv/bin/python: not found` occurs because:

1. `.barfrc` commands reference `cyrus2/.venv/bin/python` (relative path)
2. The `.venv` contents are gitignored (`cyrus2/.venv/.gitignore` has `*` rule)
3. After fresh checkout or venv deletion, the python binary doesn't exist
4. `FIX_COMMANDS` in `.barfrc` is currently empty — no setup step runs before verification
5. Previous plan marked this "ALREADY IMPLEMENTED" but issue keeps cycling — the venv happens to exist now but barf verification may run in contexts where it doesn't

## Approach

**Set `FIX_COMMANDS` to create venv + install dev dependencies before verification.**

`FIX_COMMANDS` runs best-effort before BUILD/CHECK/TEST in barf's pre-complete pipeline. Set it to:
1. Create venv if missing: `uv venv cyrus2/.venv`
2. Install dev deps: `uv pip install --python cyrus2/.venv/bin/python -r cyrus2/requirements-dev.txt`
3. Auto-fix formatting: `cyrus2/.venv/bin/python -m ruff format cyrus2/`

**Why this approach**:
- Non-invasive: only changes `.barfrc` config (one line)
- Idempotent: `uv venv` and `uv pip install` are fast no-ops when already satisfied
- Uses existing tooling (`uv` is already installed)
- Follows barf's intended pattern for pre-verification fixes
- Solves the root cause rather than just being "already working"
