## Root Cause

The error `sh: 1: cyrus2/.venv/bin/python: not found` occurs because:

1. **`barf-worker.ts` line 203** — `case 'auto'` calls `autoCommand(provider, opts, config, { db })` without passing `projectDir`
2. **`auto.ts` line 112-122** — `loopDeps` object is missing `projectCwd`, so all `runLoop` calls from auto have `projectCwd: undefined`
3. **`auto.ts` line 428** — Standalone verify calls `_verify(issue.id, effectiveConfig, provider)` with NO deps at all (no `{ cwd }`)
4. **Result**: `sh -c "cyrus2/.venv/bin/python ..."` spawns with barf-ts's process cwd, not the project directory. The relative path `cyrus2/` resolves against the wrong directory.

This affects ALL projects with custom `.barfrc` commands using relative paths when running via `barf auto`.

## Approach

Thread `projectCwd` through the auto command pipeline, matching the pattern already established in the `plan`/`build`/`verify` worker cases. This is a 4-file fix with straightforward plumbing — no new patterns needed.
