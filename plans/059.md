## Approach

**Two-part fix**:

1. **Change `.barfrc` to use absolute paths** for the Python binary. This ensures verification commands work regardless of the CWD that barf's subprocess uses.

2. **Run `ruff format`** on the 3 files with formatting issues so `CHECK_COMMAND` passes.

**Why absolute paths over alternatives**:
- The underlying barf-ts code already passes `cwd` correctly in the main code paths (`barf-worker.ts`, `outcomes.ts`, `loop.ts`), but the original error occurred when this wasn't the case. Absolute paths are a defense-in-depth fix that works regardless.
- This is a local-only development project — no portability concern.
- Plan 050 (for sibling issue) already documented this same approach.
