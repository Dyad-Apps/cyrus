# Implementation Plan: Fix projectCwd not propagated through auto command verification

**Issue**: [069](/home/daniel/Projects/barf/cyrus/issues/069.md)
**Created**: 2026-03-18
**PROMPT**: PROMPT_plan

## Gap Analysis

**Already exists**:
- `projectDir` is available in `barf-worker.ts` from `request.projectDir` for every command
- Worker `verify` command correctly passes `deps: { cwd: projectDir }` to `verifyIssue`
- Worker `plan`/`build` commands correctly pass `projectCwd: projectDir` to `runLoop`
- `runLoop` correctly propagates `projectCwd` through `OutcomeDeps` → `verifyIssue`
- `preCheckCommand()` correctly resolves relative paths when `cwd` is provided
- `.barfrc` config at `/home/daniel/Projects/barf/cyrus/.barfrc` uses correct relative paths: `cyrus2/.venv/bin/python`

**Needs building**:
- `autoCommand` in `auto.ts` does NOT receive or propagate `projectCwd`
- `AutoCommandOpts` lacks `projectCwd` field
- `loopDeps` in `autoCommand` is missing `projectCwd`
- Direct `verifyIssue` call in auto.ts verify phase passes no `deps` (no `cwd`)
- `barf-worker.ts` `auto` case doesn't pass `projectDir` to `autoCommand`
- `verifyWithSession` in orchestration.ts doesn't accept or forward `deps`

## Root Cause

The error `Command not found: cyrus2/.venv/bin/python (resolved to cyrus2/.venv/bin/python)` confirms `cwd` is `undefined` in `preCheckCommand()` — the `(resolved to ...)` suffix shows the raw `firstToken` without any cwd prefix.

**Trace**: `barf-worker.ts` `case 'auto'` → `autoCommand()` — `projectDir` is available but never passed. Inside `autoCommand`, three code paths call `runLoop` or `verifyIssue` without `projectCwd`:

1. **`loopDeps`** (line 125): No `projectCwd` property → all `runLoop` calls → `handleBuildCompletion` → `verifyIssue` with `cwd: undefined`
2. **Direct `verifyIssue`** (line 435): No `deps` → `cwd: undefined`
3. **`handleGatePhase` callbacks** (line 181-196): Spread `loopDeps` which lacks `projectCwd`

## Approach

Add `projectCwd` to the `auto` command pipeline. This is a targeted 3-file fix:

1. Add `projectCwd?: string` to `AutoCommandOpts`
2. Pass `projectDir` from `barf-worker.ts` auto case
3. Set `projectCwd` in `loopDeps` inside `autoCommand`
4. Pass `deps: { cwd: projectCwd }` to direct `verifyIssue` call
5. Fix `verifyWithSession` to accept/forward deps (currently unused but should be correct)

**Why this approach**: The `projectCwd` plumbing already exists in every other code path (plan/build/verify commands). The auto command is the only pathway that omits it. We thread it through the same way the other commands do.
