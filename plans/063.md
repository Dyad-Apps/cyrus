### Separate Bug Found (out of scope)

`preCheckCommand()` in `packages/core/src/core/verification/checks.ts` (lines 97-100) has a path resolution bug: when `cwd` is provided and the command uses an **absolute** path (e.g., `/usr/bin/python3`), it incorrectly prepends `cwd`, producing an invalid path like `/home/user/project/usr/bin/python3`. The fix is to check `firstToken.startsWith('/')` and skip `cwd` resolution for absolute paths. The existing test at line 59 tests absolute paths without `cwd` so the bug is not caught. This should be filed as a separate issue.
